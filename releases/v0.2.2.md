# FFUI v0.2.2

Full Changelog: https://github.com/muyuanjin/ffui/compare/v0.2.1...v0.2.2

## English

### Highlights

- More reliable pause/resume: stronger crash/exit recovery, steadier progress reporting, and safer concat behavior.
- Better crash recovery: queue progress hints persisted for recovery, plus snapshot revisions and event coalescing for more stable updates.
- Proxy & updater improvements: SOCKS5 proxy URL support, proxy-failure fallback to direct, and updater proxy overrides with persisted settings.
- Tooling & maintenance: pnpm migration, expanded checks/audits, and a set of backend hardening and clippy cleanups.

### Details

- Engine & reliability
  - Improves pause/resume reliability across crash/exit recovery, including segment target tracking and rollback.
  - Stabilizes boundary metadata and avoids truncation on rapid pause/resume.
  - Improves stderr draining around ffmpeg exit and hardens resume cleanup/trim logic.
- Queue & state
  - Persists ffmpeg progress hints for crash recovery; refreshes after resume and dedupes snapshots.
  - Adds snapshot revisioning and coalesces queue events to reduce UI churn.
  - Stabilizes reorder behavior and filter parsing; shares preset lookup helpers and precomputes batch counts.
- UI & i18n
  - Ensures select trigger labels render reactively on locale switch.
  - Refines import dialog flow and skipped navigation; tightens UI/state sync tests.
  - Shows app version in the titlebar.
- Tools, network & updates
  - Adds SOCKS5 proxy URL support and falls back to direct when proxy failures occur.
  - Stabilizes tool download status, update-check isolation/logging, and unknown-update surfacing in the UI.
  - Passes updater proxy override and persists updater settings.
- Platform & backend hardening
  - Hardens Tauri backend against panics; disables `withGlobalTauri` by default.
  - Improves Windows Job Object handle management and leak prevention.
  - Tightens lock scopes and reduces clones across backend sync paths.
- Tooling, checks & deps
  - Migrates to pnpm; removes deprecated husky bootstrap.
  - Adds/updates jscpd duplicate-code checks and “check-all” scripts; bumps toolchain and adds audit helpers.

## 中文

### 重点更新

- 暂停/续传更稳：增强崩溃/退出恢复能力，进度统计更稳定，并降低 concat 异常风险。
- 崩溃恢复更好：队列恢复会持久化 ffmpeg 进度提示；新增快照修订号与事件合并，更新更稳定。
- 代理与更新改进：支持 SOCKS5 代理 URL；代理失败可回退直连；更新器支持代理覆盖并持久化配置。
- 工程化与维护：迁移至 pnpm，扩展检查/审计脚本，并补齐后端加固与 clippy 清理。

### 细节说明

- 引擎与稳定性
  - 强化暂停/续传在崩溃/退出场景下的恢复能力（包含 segment target 追踪与回滚）。
  - 稳定 boundary metadata，并避免快速暂停/续传导致的 concat 截断。
  - 改善 ffmpeg 退出前后的 stderr drain，并强化续传清理/trim 逻辑。
- 队列与状态
  - 为崩溃恢复持久化 ffmpeg 进度提示；续传后刷新并去重快照。
  - 新增快照修订号并合并队列事件，减少 UI 抖动。
  - 稳定拖拽重排与过滤解析；复用预设查找辅助函数并预计算批次数量。
- 界面与多语言
  - 修复下拉触发器已选项文本在切换语言后不立即更新的问题。
  - 优化导入弹窗与“跳过任务”的导航；补齐 UI/状态同步测试。
  - 标题栏展示应用版本号。
- 工具、网络与更新
  - 支持 SOCKS5 代理 URL；代理失败时回退直连。
  - 稳定工具下载状态、更新检查隔离/日志，并在 UI 中提示未知更新检查结果。
  - 更新器支持代理覆盖，并持久化更新器设置。
- 平台与后端加固
  - 加固 Tauri 后端防 panic；默认禁用 `withGlobalTauri`。
  - 改善 Windows Job Object 句柄管理，避免并发初始化泄漏。
  - 收紧锁范围并减少后端同步路径上的不必要拷贝。
- 工程化、检查与依赖
  - 迁移至 pnpm；移除过期 husky bootstrap。
  - 增加/更新 jscpd 重复代码检查与 check-all 脚本；升级工具链并加入审计辅助脚本。
